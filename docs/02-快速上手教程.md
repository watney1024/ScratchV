## 1. Hello World: 你的第一个 DSL 程序

ScratchV 有自己的迷你编程语言（DSL），用来描述张量计算。我们先写一个最简单的：

### 1.1 创建 DSL 文件

```bash
# 在项目根目录下
mkdir -p my_examples
cat > my_examples/hello.dsl << 'EOF'
# 我的第一个 DSL 程序: 两个数相加
a = const(3)
b = const(5)
c = add(a, b)
return c
EOF
```

### 1.2 编译为 IR（中间表示）

DSL 程序第一步是翻译成 ScratchV 的中间表示（IR）：

```bash
python3 -c "
from scratchv.frontend.dsl_parser import DSLParser
parser = DSLParser()
program = parser.parse_file('my_examples/hello.dsl')
program.print()
"
```

预期输出：
```
Function main:
  %0 = const 3
  %1 = const 5
  %2 = add %0, %1
  ret %2
```

这就是 **三地址码 IR**。每一行是一个操作，最多三个参与者（两个输入，一个输出）。

### 1.3 编译为 RISC-V 汇编

```bash
python3 -c "
from scratchv.frontend.dsl_parser import DSLParser
from scratchv.backend.instruction_select import InstructionSelector
from scratchv.backend.register_alloc import RegisterAllocator
from scratchv.backend.asm_emit import AsmEmitter

# 解析 DSL
parser = DSLParser()
program = parser.parse_file('my_examples/hello.dsl')

# 指令选择 + 寄存器分配 + 汇编发射
selector = InstructionSelector()
instrs = selector.select(program)

allocator = RegisterAllocator()
allocated = allocator.allocate(instrs)

emitter = AsmEmitter()
asm = emitter.emit(allocated)
print(asm)
"
```

预期输出（类似）：
```asm
main:
    li a0, 3
    li a1, 5
    add a2, a0, a1
    mv a0, a2
    ret
```

> 💡 这就是你写的 `c = add(a, b)` 变出来的 RISC-V 汇编。`li` = 加载常数，`add` = 加法，`mv` = 移动数据，`ret` = 返回结果。

## 2. 理解编译产物

### 2.1 指令长什么样？

RISC-V 汇编的格式是：`操作码 目标寄存器, 源寄存器1, 源寄存器2`

```asm
add a2, a0, a1    ←  a2 = a0 + a1
mul a3, a0, a1    ←  a3 = a0 * a1
lw  a4, 0(a0)     ←  a4 = 从内存地址 a0 加载 32 位数据
sw  a4, 0(a0)     ←  把 a4 存到内存地址 a0
```

寄存器名字 `a0-a7` 是函数参数/返回值，`t0-t6` 是临时变量，`s0-s11` 是保留变量。

### 2.2 编译流程全景

```
my_examples/hello.dsl     ← 你写的 DSL 代码
        │
   DSLParser               ← 解析成 AST
        │
   IRBuilder               ← 生成 IR（三地址码）
        │
   [Optimizer]             ← (可选) 优化 IR
        │
   InstructionSelector     ← IR → RISC-V 伪指令
        │
   RegisterAllocator       ← 虚拟寄存器 → 物理寄存器
        │
   AsmEmitter              ← 输出汇编文本
        │
   RISC-V 汇编 (.s)         ← 人类可读
        │
   RISC-V Encoder          ← 汇编 → 32-bit 机器码
        │
   output.bin              ← CPU 可直接执行
```

## 3. 编译 CNN 模型

ScratchV 最核心的功能是把 ONNX 格式的 CNN 模型编译成 RISC-V 机器码。

### 3.1 一条命令编译

```bash
make bench-cnn
```

这条命令背后执行的是：

```bash
python3 scratchv/standalone/onnx_to_riscv_standalone.py models/graph/cnn.onnx \
    -o /tmp/cnn_riscv.bin --estimate --report
```

### 3.2 参数说明

| 参数 | 作用 |
|------|------|
| `models/graph/cnn.onnx` | 输入：ONNX 模型文件 |
| `-o /tmp/cnn_riscv.bin` | 输出：RISC-V flat binary |
| `--estimate` | 开启性能估算（动态指令数） |
| `--report` | 生成详细报告（Markdown 格式） |
| `--asm output.s` | (可选) 同时输出汇编文件 |

### 3.3 查看输出

```bash
# 查看汇编输出（如果加了 --asm）
head -100 output.s

# 查看性能报告
cat benchmark_reports/cnn_riscv_report.md
```

报告内容示例：
```markdown
| 层 | 静态指令 | 动态指令 | MAC 数 |
|----|---------|---------|--------|
| Conv1 | 245 | 1,234,567,890 | 110,592,000 |
| Conv2 | 245 | 884,736,000 | 73,728,000 |
| Conv3 | 167 | 1,105,920,000 | 92,160,000 |
| MaxPool1 | 48 | 12,582,912 | — |
| ... | ... | ... | ... |
| **总计** | **749** | **3,222,000,000** | — |
```

> 💡 **静态指令** = 编译器生成了多少行指令（不管循环执行多少次）。**动态指令** = CPU 实际执行了多少条（循环里的指令每次迭代都算），这个数才是真正影响运行速度的。

## 4. 用 LLVM 路径编译

ScratchV 还有一条 LLVM 路径，生成 LLVM IR 而不是直接生成 RISC-V 机器码。

```bash
# 生成 LLVM IR（需要先 pip install llvmlite）
python3 scratchv/standalone/onnx_to_llvm_standalone.py models/graph/cnn.onnx \
    -o output.ll --compare
```

`--compare` 会让它估算两条路径的动态指令数并对比。

LLVM IR 长这样（示例片段）：
```llvm
; Conv2D 内层循环
%in_ptr = getelementptr float, float** %input, i32 %in_offset
%in_val = load float, float* %in_ptr
%w_val = load float, float* %w_ptr
%mul = fmul float %in_val, %w_val
%acc = fadd float %acc, %mul
```

对比 ScratchV 原生路径（Q16.16 定点）：
```asm
# 相同逻辑，但用整数指令模拟浮点
lw t0, 0(a0)          # load input (Q16.16)
lw t1, 0(a1)          # load weight (Q16.16)
mul t2, t0, t1        # 乘法 → 64-bit
srai t2, t2, 16       # 右移16位还原定点
add a2, a2, t2        # 累加
```

你会注意到：**LLVM 用 `fmul` (浮点乘) 一条指令搞定了 ScratchV 的 `mul + srai` 两条**。

## 5. 运行基准测试

```bash
# 基础功能测试
make test

# CNN 模型编译 + 估算
make bench-cnn

# 完整 CI 基准（LLVM + TinyFive + 仪表盘）
make bench-ci
```

各命令耗时参考：

| 命令 | 耗时 | 产出 |
|------|------|------|
| `make test` | ~1 分钟 | 终端测试结果 |
| `make bench-cnn` | ~5 秒 | 性能报告 .md |
| `make bench-ci` | ~2 分钟 | 仪表盘 .html + JSON + MD 报告 |

## 6. 查看性能仪表盘

```bash
# 生成仪表盘
make bench-dashboard

# 用浏览器打开
# (生成的 HTML 是纯静态的，零依赖)
firefox benchmark_reports/dashboard.html
# 或
google-chrome benchmark_reports/dashboard.html
```

仪表盘展示：
- **静态指令数对比**：ScratchV vs LLVM，按算子拆分
- **动态指令数对比**：带百分比差异
- **缓存命中率**：D-Cache 和综合评分
- **趋势图**（history.html）：历次提交的性能变化曲线
