# 01 基本块（Basic Block）

## 什么是基本块？

基本块是程序中**只能从第一条指令进入、只能从最后一条指令退出**的直线指令序列。块内没有分支，也没有跳转目标——所有指令按顺序一口气执行完。

```
# 这是一个基本块 —— 直线执行，没有分支
add $v1, $a, $b
sub $v2, $v1, $c
mul $v3, $v2, $d

# 下面出现了分支，此处开始新块
br_if $cond -> .Lthen, .Lelse
```

> **直觉理解**：基本块就是一段"不拐弯"的代码——进去之后一定会执行到块尾，中间不会跳到别处去。

## 块边界规则

在 ScratchV IR 中，基本块的分割由以下指令决定：

| 指令 | 作用 | 说明 |
|------|------|------|
| `LABEL` | **开始**新块 | 遇到 LABEL 标记，新建一个块 |
| `BR` | **结束**当前块 | 无条件跳转，1 个出口 |
| `BR_IF` | **结束**当前块 | 条件分支，2 个出口 |
| `RETURN` | **结束**当前块 | 函数返回，0 个出口 |
| `FOR` | **结束**当前块 | 循环入口，2 个出口 |
| `ENDFOR` | **结束**当前块 | 循环出口，1 个出口（跳回 FOR） |

**核心原则**：每个基本块最多有一条终止指令（terminator）。如果块末尾没有终止指令，它就会自然"落到"（FALLTHROUGH）下一个块。

## ASCII 图解：if-else 程序

下面是一个 if-else 程序被分割成基本块的过程：

```
原始 IR 指令流               →      基本块划分

.Lentry:                      ┌──────────────────┐
  br_if $cond -> .Lthen,.Lelse│     entry        │
.Lthen:                       │  br_if $cond     │
  add $v1, $x, $y            └───┬──────────┬────┘
  br -> .Lend                    │ true      │ false
.Lelse:                     ┌────▼───┐ ┌───▼──────┐
  sub $v2, $x, $y           │  then  │ │  else    │
  br -> .Lend               │  add   │ │  sub     │
.Lend:                      │  br    │ │  br      │
  ret $result               └───┬────┘ └───┬──────┘
                               │           │
                               └─────┬─────┘
                              ┌──────▼──────┐
                              │    end      │
                              │   ret       │
                              └─────────────┘
```

> 每个框就是一个基本块。箭头代表控制流转移的方向——实线可以想象成"无条件走过去"，虚线是"根据条件选择一条路"。

## 常见陷阱

### 1. FALLTHROUGH vs JUMP
没有终止指令的块会自然**落到**下一个块（FALLTHROUGH）。而 `BR` 是**显式跳转**（JUMP）。不要搞混——FALLTHROUGH 是顺序执行、不加指令的；JUMP 是一条实际的跳转指令。

### 2. 连续 LABEL 产生空块
```
.La:             # 块 A
.Lb:             # 块 B（空！没有指令）
  add $v1, $x, $y
```
`La` 和 `Lb` 之间会产生一个空的基本块。编译器前端一般不会生成这样的 IR，但分析器必须正确处理。

### 3. 终止指令之后的指令
```
  ret $v1         # 块结束
  add $v2, $x, $y # 这条指令不会被执行
```
`RETURN` 之后的指令虽然不会被执行，但技术上仍然属于同一个基本块。我们的分析器不会自动分割它们——检测不可达代码是后续（Todo 7）的工作。

## 小结

基本块是 CFG 的"原子单位"。把线性指令流切割成基本块，是构建控制流图的第一步，也是最关键的一步。接下来（Todo 2-3）我们会把块之间的边补全，形成完整的 CFG 数据结构。
