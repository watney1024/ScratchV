# Topic 21 — IR 验证器

> **难度**: 高级 | **源文件**: `scratchv/analysis/ir_verifier.py`

---

## 是什么？

IR 验证器检查 ScratchV IR Program 是否符合**正确性规则**。它应该在每次优化 pass 前后运行——确保优化没有破坏 IR。

验证 7 条规则：
1. **def-before-use**: 所有操作数必须在使用前定义
2. **label-existence**: 跳转目标标签必须存在
3. **block-termination**: 每个基本块必须以终止指令结束
4. **type-consistency**: 运算操作数类型必须兼容
5. **control-flow-integrity**: 不可达代码检测
6. **ssa-validity**: 每个变量只赋值一次
7. **entry-existence**: 函数必须有至少一个基本块

---

## 为什么？

优化 pass 是 IR→IR 的变换，很容易引入错误：
- 死代码消除可能误删还在使用的变量
- 常量折叠可能改变类型
- 循环不变量外提可能破坏 SSA

验证器是**安全网**——在 IR 进入后端（产生不可逆的机器码）之前，拦截所有结构性问题。

---

## 核心概念

### 错误级别

| 级别 | 含义 |
|------|------|
| **ERROR** | 确定会导致编译失败或运行时错误 |
| **WARNING** | 可能有问题但不一定（如类型不匹配） |

### VerificationError 结构

```python
@dataclass
class VerificationError:
    level: ErrorLevel           # ERROR 或 WARNING
    message: str                # 人类可读描述
    function_name: str | None   # 所在函数
    block_name: str | None      # 所在基本块
    instruction_index: int | None  # 指令索引
    value_name: str | None      # 有问题的值
    rule: str | None            # 规则标识符
```

## 一步步

### 在编译管线中使用

```python
from scratchv.analysis.ir_verifier import IRVerifier

# 优化前验证
pre_errors = IRVerifier(program).verify()
assert not pre_errors, f"IR invalid: {pre_errors}"

# 运行优化
run_optimizations(program)

# 优化后验证
post_errors = IRVerifier(program).verify()
assert not post_errors, f"Optimization broke IR: {post_errors}"
```

### 快速验证

```python
from scratchv.analysis.ir_verifier import verify_ir

passed, errors = verify_ir(program)
if not passed:
    for err in errors:
        print(f"[{err.level.name}] {err.message}")
```

## 代码走读

### def-before-use 检查

```python
def _check_def_before_use(self, func):
    defined = set()  # 已定义的变量
    for block in func.blocks:
        for i, instr in enumerate(block.instructions):
            for op in instr.operands:
                if op.name not in defined and not op.constant:
                    self._add_error(ErrorLevel.ERROR,
                        f"value '{op.name}' used before definition",
                        func_name=func.name, block_name=block.name,
                        instruction_index=i, value_name=op.name,
                        rule="def-before-use")
            if instr.dest:
                defined.add(instr.dest.name)
```

### SSA 检查

```python
def _check_ssa(self, func):
    assigned = set()
    for block in func.blocks:
        for instr in block.instructions:
            if instr.dest:
                if instr.dest.name in assigned:
                    self._add_error(ErrorLevel.ERROR,
                        f"value '{instr.dest.name}' assigned multiple times",
                        rule="ssa-validity")
                assigned.add(instr.dest.name)
```

### block-termination 检查

每个基本块的**最后一条指令**必须是终结指令（RETURN, BR, BR_IF 之一），否则控制流会"掉进"下一个块——虽然是合法的，但通常是 bug。

## 常见坑

| 坑 | 说明 |
|----|------|
| **常量豁免** | 常量（`value.constant == True`）不需要 def-before-use 检查——它们天然"已定义" |
| **副作用的终结指令** | RETURN/BR/BR_IF 后的指令是不可达的，但不是所有 pass 都会清理 |
| **验证器性能** | 大型 IR（如 CNN 展开后的全循环 IR）验证可能较慢，开发时可选择性跳过 |
