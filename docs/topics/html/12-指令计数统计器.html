Topic 12 — RISC-V 后端指令计数统计器

难度: 入门 | 源文件: scratchv/backend/inst_counter.py | 行数: ~608

是什么？

指令计数统计器（inst_counter.py）解析 RISC-V 汇编文件，按类别统计指令数量，并生成文本表格、matplotlib 图表和 HTML 报告。

统计类别：

类别 包含的指令 例子
ALU 算术/逻辑运算 add, sub, mul, srai, and, or
MEM 内存读写 lw, sw, lh, sh, flw, fsw
BRANCH 条件分支 beq, bne, blt, bge
JUMP 无条件跳转 j, jal, jalr, ret
PSEUDO 伪指令（汇编语法糖） li, mv, nop, call
MISC 其他/未知 无法归类的指令

为什么？

编译器优化的核心指标之一就是指令数。你需要知道：

  1. 各类指令占比：ALU 多 = 计算密集，MEM 多 = 数据密集
  2. 优化前后对比：优化到底减少了多少条指令？哪一类减少最多？
  3. ScratchV vs LLVM：两个编译器的指令分布有什么不同？

没有这个工具，你只能手动数——费时还容易出错。

核心概念

1. 指令分类字典

_OPCODE_CATEGORIES = {
    "add": "ALU", "sub": "ALU", "mul": "ALU", ...
    "lw": "MEM", "sw": "MEM", ...
    "beq": "BRANCH", "bne": "BRANCH", ...
    "j": "JUMP", "jal": "JUMP", ...
    "li": "PSEUDO", "mv": "PSEUDO", ...
}

每条 RISC-V 操作码（opcode）映射到一个类别。

2. 三类输出格式

格式 用法 适用场景
文本表格 format_table(counts) 终端快速查看
matplotlib 图表 generate_chart(counts, "chart.png") 报告、演示
HTML 报告 generate_html_report(counts, "report.html") CI 产物、Dashboard

3. 多文件对比

from scratchv.backend.inst_counter import compare_files

result = compare_files(["scratchv_output.s", "llvm_output.s"])
print_comparison(result)  # 并排对比表

python -m scratchv.backend.inst_counter scratchv.s llvm.s --compare --chart comparison.png

代码走读

指令提取

def _extract_opcode(line: str) -> Optional[str]:
    stripped = line.strip()
    # 跳过空行和注释
    if not stripped or stripped.startswith("#"):
        return None
    # 去掉行尾注释
    code_part = stripped.split("#")[0].strip()
    # 跳过硬编码的汇编器指令
    if any(code_part.lower().startswith(d) for d in _DIRECTIVES):
        return None
    # 提取第一个 token → 操作码
    tokens = code_part.replace(",", " ").split()
    first = tokens[0].lower()
    # 跳过标签（以 : 结尾）
    if first.endswith(":"):
        return None
    return first

分类逻辑

def _classify_opcode(opcode: str) -> str:
    return _OPCODE_CATEGORIES.get(opcode, "MISC")  # 未知 → MISC

对比功能

def compare_files(filepaths):
    # 以第一个文件为 baseline
    baseline = counts[files[0]]
    for f in files[1:]:
        diffs[f] = {}

            diffs[f][cat] = counts[f][cat] - baseline[cat]

动手练习

练习 1: 分析 CNN 模型的指令分布

用 make bench-cnn 生成汇编，统计各类指令占比。哪类最多？为什么？

练习 2: 优化前后对比

对同一段代码，分别统计优化前和优化后的指令分布，计算减少了多少条、哪些类别变化最大。

练习 3: 添加新的指令分类

在 _OPCODE_CATEGORIES 中添加 RISC-V F/D 扩展（浮点）指令的映射。

常见坑

坑 说明
伪指令 vs 真指令 li a0, 5 实际上可能被汇编器展开为 addi a0, x0, 5，统计的是展开前还是展开后？这里统计的是汇编源文件中的操作码
自定义指令 ScratchV 的伪指令（如 max）映射到 ALU 类别，如果添加了新指令记得更新 _OPCODE_CATEGORIES
matplotlib 依赖 生成图表需要 pip install matplotlib，但不是必需功能
指令识别遗漏 操作码提取依赖正则匹配，某些特殊格式的汇编行可能漏掉（检查 _INST_RE 正则是否覆盖）
